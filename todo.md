현재 구조

유니티 <-> 컨트롤러

컨트롤러 <-> 각 컨테이너

1) 음성 입력 -> 파일화 -> 컨트롤러 -> stt -> c -> llm -> c -> tts
2) 문자 입려 -> 컨트롤러 -> llm -> c -> tts

현재 딜레이 0.1~0.4초 사이

각 기능은 컨테이너화 되어있으며,
컨트롤러에서 웹소켓 라우터를 관리하여 기능 추가 및 제외가 용이하게 설계되어있음.
각 컨테이너에서는 컨트롤러와 데이터를 송수신 하는 함수를 제외하고는
새로이 함수, 기능을 추가할 수 있음
